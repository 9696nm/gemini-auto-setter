# Gemini Auto Setter

Gemini (https://gemini.google.com/) を開いた際に、**一時チャット**と**指定モード**（思考モードなど）を自動で適用する Chrome 拡張機能です。

**リポジトリ**: [github.com/9696nm/gemini-auto-setter](https://github.com/9696nm/gemini-auto-setter)

## 機能

- **一時チャットの自動選択**: 新しいチャットを開く際に「一時チャット」を自動で選択
- **モードの自動選択**: 思考モード、標準、実験的など、任意のモードを自動で選択
- **トリガーURLのカスタマイズ**: どのURLパスで自動適用するかを設定可能

## インストール

1. リポジトリをクローンする
   ```bash
   git clone https://github.com/9696nm/gemini-auto-setter.git
   ```
2. Chrome で `chrome://extensions/` を開く
3. 「デベロッパーモード」を有効にする
4. 「パッケージ化されていない拡張機能を読み込む」をクリック
5. クローンした `gemini-auto-setter` フォルダを選択

## 使用方法

1. **インストール**後、Chrome の拡張機能バーに Gemini Auto Setter のアイコンが表示されます
2. 拡張機能アイコンを**クリック**、または**右クリック** → 「オプション」で設定画面を開く
3. 必要な設定を行う（一時チャットの自動選択、モード選択など）
4. 「保存」をクリック
5. **https://gemini.google.com/** を開く（または既に開いている場合はリロード）
6. トリガーURLにマッチするページで、一時チャットとモードが自動で適用されます

> 初回や SPA の描画完了を待つため、数秒かかることがあります。オプションの「実行遅延」で待機時間を調整できます。

## 設定

拡張機能のアイコンを**クリック**、または右クリック → 「オプション」、または `chrome://extensions/` から「詳細」→「拡張機能のオプション」で設定画面を開けます。

### 設定項目

| 項目 | 説明 |
|------|------|
| 拡張機能を有効にする | オフにすると自動適用を停止 |
| 一時チャットを自動で選択する | 一時チャットの自動選択の有無 |
| モード | 思考モード / 標準 / 実験的 / なし |
| トリガーURL | 1行に1つ、マッチさせるURLパスを入力。例: `/u/1/app`, `/app`, `/u/1/app?pli=1` |

### トリガーURLの例

- `/u/1/app` - 特定のパス
- `/app` - アプリのルート
- `/u/1/app?pli=1` - クエリ付き
- 空欄 - 常に適用

## 動作しない場合のフィードバックの送り方

拡張機能が動作しないときは、以下の手順で情報を集めて共有してください。

### 1. デバッグモードを有効にする

1. 拡張機能のオプション画面を開く
2. **「デバッグモード（コンソールに詳細ログを出力）」** にチェックを入れる
3. 「保存」をクリック

### 2. コンソールログを取得する

1. https://gemini.google.com/ を開く（ログイン済みの状態で）
2. **F12** キーで開発者ツールを開く
3. **Console** タブを選択
4. ページをリロード（F5）する
5. `[Gemini Auto Setter]` で始まるログを探す

### 3. 共有してほしい情報

以下の内容をコピーして送ってください。

- **コンソールログ**: `[Gemini Auto Setter]` のログ全文
- **現在のURL**: アドレスバーに表示されているURL（例: `https://gemini.google.com/u/1/app`）
- **どの部分が動かないか**: 一時チャット / モード選択 / 両方
- **ブラウザの言語**: 日本語 / 英語 など

### 4. （任意）DOM情報の取得

一時チャットやモード選択のボタンが見つからない場合、次の情報があると修正しやすいです。

1. 一時チャットのボタンやリンクを **右クリック** → 「検証」
2. 開いた Elements パネルで、該当要素の `class`、`aria-label`、`role` などの属性を確認
3. その属性値や、親要素の構造をメモして共有

## 技術メモ

- **メニューを開く必要**: モード選択肢（思考モード、高速モードなど）はオーバーレイ内にあり、メニューを開くまで DOM に存在しません。そのため、直接 `data-mode-id` で設定することはできず、必ず「モデルセレクターを開く → 該当オプションをクリック」の流れになります。
- モードは `data-test-id`（例: `bard-mode-option-思考モード`）で正確に識別しています。

## 注意事項

- Gemini の UI は Google の更新で変更されることがあります。その場合、正しく動作しなくなる可能性があります。
- Shadow DOM 内の要素を操作するため、DOM 構造が大きく変わると修正が必要になる場合があります。
